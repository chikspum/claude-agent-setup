# Active Plans

Store active execution plans here while the work is still open.

Use one markdown file per non-trivial task.

Recommended lifecycle:

1. create from [TEMPLATE.md](/home/ubuntu/claude-agent-setup/docs/plans/active/TEMPLATE.md)
2. give the plan a stable task id and owner
3. update milestone status and validation notes as work progresses
4. move the finished plan into `docs/plans/completed/`
5. leave a validation artifact in `artifacts/validations/` when the task is substantial

Plans are working artifacts, not polished documents.

Naming convention:

- `YYYY-MM-DD-short-topic.md` for one-off work
- `phase-x-y-topic.md` for roadmap-aligned work that stays open across sessions
