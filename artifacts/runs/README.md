# Run Logs

Store one markdown file per hybrid execution attempt that is worth keeping.

Use run logs to capture:

- which plan or task was executed
- whether Claude Code or Codex performed the work
- commands that actually ran
- missing-tool or environment blockers
- whether follow-up validation or rework was required

Recommended naming:

- `YYYY-MM-DD-<task-id>-run-01.md`
- increment the run suffix for repeated attempts on the same task

Run logs are lightweight operational records.
Validation artifacts remain the acceptance record.
